# 数据使用规范

> 这套规则是从实际报告迭代中沉淀的，**违反任何一条都会让报告失去可信度**。

---

## Rule 1：所有数字必须可在 PDF 原文中找到出处

### 红线
- ❌ 不可凭推测/常识/记忆填数字
- ❌ 不可从研报、新闻、行业平均推算
- ❌ 不可"约""大概""估计"模糊数字
- ✅ 要么写精确值，要么写 "_待补_"

### 推荐工作流
1. 用 `pdfplumber` 完整提取财报 PDF 所有页文本
2. 关键数据先**搜索关键词**定位页码，再**读取整页**核对上下文（避免行错位）
3. 写报告时，**所有正文数字单独列一遍 → 逐个回查 PDF**

### 常见陷阱

**陷阱 1：PDF 表格列错位**
财务报表有大量数字密集的表格，pdfplumber 提取时可能把数字"挤"到错误的列。例如：
```
公司 总资产 净资产 营业收入 营业利润 净利润
A    100    50    20      -      -3      -5
```
PDF 抽取后可能变成：
```
A 100 50 20 - -3 -5
```
这里 `-3` 是营业利润 -300 万，`-5` 是净利润 -500 万，不能误读为"营业利润 -3 万"。

**对策**：遇到表格中带 `-` 的数字，必须读整行原始文本上下文，确认列对应。

**陷阱 2：单位混淆**
财报常用"元"，但精读版要换成"万元/亿元"。换算时核对小数点位置：
- 1,234,567,890 元 = 12.35 亿元 ≠ 1.23 亿元
- 12,345,678 元 = 1,234.57 万元 ≠ 12,345.68 万元

**陷阱 3：年度数据 vs 季度数据**
某些指标（同比、环比）容易混淆来源期间。在表格中**必须明确标注期间**：
- ✅ "2026Q1 销售收现 vs 2025Q1 销售收现"
- ❌ "销售收现 vs 上期"

## Rule 2：同比 vs 环比的选用

### 通用原则

| 数据类型 | 优先用 | 原因 |
|---|---|---|
| 利润表科目（流量数）| **同比** | 受季节性影响 |
| 现金流量表科目（流量数）| **同比** | 受季节性影响 |
| 资产负债表科目（时点数）| **环比** | 无季节性，环比反映动量 |

### 例外情况
- 季节性极强的公司（如 Q4 占全年 50%+）：必须用同比避开季节性误导
- 周期性行业：用 3-5 年周期内位置，单纯同比环比都不够
- 重大事件期（IPO、并购、剥离）：要做"模拟可比口径"再比较

### 报告里必须写的"使用约定"
在报告开头明确告诉读者你的用法，避免读者疑惑。例：

> **关于"同比 vs 环比"的使用约定**：本公司收入季节性较强，本报告采用如下原则：
> - 利润表 / 现金流表科目 → 以**同比**（vs 上年同期）为主
> - 资产负债表科目 → 以**环比**（vs 上期末）为主

## Rule 3：单位换算与精度

### 标准换算
- 1 亿 = 100,000,000 元 = 10,000 万元
- 1 万 = 10,000 元

### 精度规则
- 金额：亿级保留 2 位小数（1.79 亿元），万级保留整数或 1 位小数（1,932 万元）
- 百分比：保留 1-2 位小数（69.33% / 36.3%）
- 比率：保留 2 位小数（1.77 倍）
- 同比/环比变化率：> 100% 不写小数（+272%），< 100% 保留 1 位小数（+14.0%）

### 计算后必须复核
所有派生指标（毛利率、收现率、有息负债率等）写完后，自己用计算器走一遍：
- 毛利率 = (营收 - 营业成本) ÷ 营收
- 收现率 = 销售商品收到的现金 ÷ 营业收入
- 资产负债率 = 总负债 ÷ 总资产

## Rule 4：行业警戒线速查

### 利润现金保障倍数（经营现金流 / 净利润）
| 区间 | 信号 |
|---|---|
| > 1.0 | 🟢 优秀 |
| 0.5 - 1.0 | 🟡 正常 |
| 0 - 0.5 | 🔴 警惕 |
| < 0（亏损时不适用） | 用绝对值对比 |

### 销售变现率（销售收现 / 营收）
含税收现 / 不含税营收，所以理论健康值 ≈ 1.13

| 区间 | 信号 |
|---|---|
| > 1.15 | 🟢 优秀 |
| 1.00 - 1.15 | 🟢 正常 |
| 0.85 - 1.00 | 🟡 偏弱 |
| < 0.85 | 🔴 警惕 |

### 财务费用率（财务费用 / 营收）
| 区间 | 制造业 | 消费/科技 |
|---|---|---|
| < 2% | 🟢 | 🟢 |
| 2% - 5% | 🟢 | 🟡 |
| 5% - 8% | 🟡 | 🔴 |
| > 8% | 🔴 | 🔴🔴 |

### 资产负债率
| 行业 | 健康区间 |
|---|---|
| 制造业 | 50% - 65% |
| 消费品 | 30% - 50% |
| 科技/平台 | < 40% |
| 房地产/建筑 | 70% - 85%（剔除预收款后看真负债率）|

### 资本开支占营收比
| 区间 | 信号 |
|---|---|
| > 20% | 🟢🟢 激进扩张 |
| 5% - 20% | 🟢 正常投入 |
| < 5% | 🟡 维持性投入 |
| 断崖式下降 | 🔴 可能放弃扩张 |

## Rule 5：交叉验证清单

报告写完后，按下面这个清单**逐项核对**：

- [ ] 红绿灯表中的每个判断 → 与第二章正文叙述一致
- [ ] 第二章 6 题中每个数字 → 在 PDF 原文中能找到
- [ ] 每个比率（毛利率、收现率等）→ 用原始数字算一遍能对上
- [ ] 同比/环比标注 → 与数据来源期间一致
- [ ] ASCII 闭环图中的每个数字 → 与表格里的数字一致
- [ ] 风险章节的金额、占比 → 与正文/附录一致
- [ ] 附录数据底稿 → 与正文引用的数字一致

**任何一项对不上，必须立即修正，不可"差不多就行"。**

---

## Rule 6：诚实优先

如果发现自己之前犯了错（比如数字错了、判断错了）：
- ✅ 立即承认，明确指出错在哪
- ✅ 用 PDF 原文给出正确数据
- ✅ 反思错因，避免重犯
- ❌ 不要辩护、不要找借口、不要"基本正确"

数字精度上的诚实，是这份报告的全部信用基础。
